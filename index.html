Remote Device Access

TCP/IP Socket Programming

Remote Device Access(RDA) consists of three major components: TCPServer, RemoteDevice and TCPClient. TCPServer is always on the listening mode to accept connections from both RemoteDevice and TCPClient. RemoteDevice is always connected to the server and awaits for the requests from the client, which is relayed to it by the server and responds to them accordingly. TCPClient connects to the server to request the whereabouts of the RemoteDevice, there are three possible requests: list, transfer, location.

  • List

The list command displays the list of all files available at the provided path of the RemoteDevice(as an argument).

  • Transfer

The transfer command saves a file physically from RemoteDevice to the TCPClient. The path at which the file is save can be modified accordingly in the code, defaulted to ~/files/ folder. The file name and path has to be given as additional arguments.

  • Location

The location command provides the exact location of the RemoteDevice to the TCPClient.

Session Initiation Protocol Analysis

Application Layer Signaling Protocol

Voice over internet protocol (VoIP) service uses IP (Internet protocol) phone to communicate over internet or LAN (Local area network). VoIP technology has gained immense user interest due to free and affordable International calls at low rates. RTP (Real-time transport protocol) is used to transmit voice data between source and destination using IP address and port address over internet. Session Initiation Protocol (SIP) was invented to help RTP find destination IP address and port address over internet. These days IP calling has become popular via applications like Skype, Viber, Whatsapp, etc. VoIP uses a signaling protocol like SIP to process a call and a media transfer protocol like RTP for voice transmission.
We analysed SIP using WireShark packet analyzer software, Open Source SIP (OpenSIPS) as proxy/server and Zoiper as external SIP based VoIP application to make VoIP calls.

Layout OCR

Digitizing typewritten/printed text - Accenture Product

Optical character recognition is the mechanical or electronic conversion of scanned or photographed images of typewritten or printed text into machine-encoded/computer-readable text. Layout OCR is the process of identifying and categorizing the reagions of interest in the scanned image of a text document. A reading system requires the segmentation of text zones from non-textual ones and the arrangment in their correct reading order.

Layout OCR provides four features; Single File Conversion: This feature allows the users to upload single file to the application and get the OCRed file on-fly! Zonal Image Capture: This feature allows the users to process a region\part of the image. Also, these settings can be saved for future usage! Batch File Upload: This feature allows the users to upload multi files in zip format. These uploaded files get processed in the backend by schedule job. Upon completion the users can download via Batch File Download feature! Batch File Download: This feature allows the users to download the processed files uploaded via Batch File Upload feature!
